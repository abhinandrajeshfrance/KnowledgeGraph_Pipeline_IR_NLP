What is reasoning
• A reasoner infers knowledge that is not explicitly stated in the 
ontology.
• Think of OWL as a structured knowledge base.
A reasoner is the “logic engine” that derives hidden consequences.
• Two kinds of reasoning
• Terminological Reasoning (TBox reasoning): Reasoning about classes 
and hierarchy
• Instance Reasoning (ABox reasoning): Reasoning about individuals

Motivation: Why Do We Need Rules?
• OWL is powerful — but not sufficient for all modeling needs.
• Limitations of OWL
• Cannot easily express certain conditional logic
• Limited expressiveness for composite relations
• Designed for decidability (so some expressive power is sacrificed)
• Example of the limitation: 
• We want to express: “If a person has a parent who has a brother, then 
that brother is the person’s uncle.”
• This is difficult in pure OWL. But easy with rules:
8
hasParent(x,y) ∧ hasBrother(y,z) → hasUncle(x,z)

===== PAGE 9 =====
Write a rule
• Rules are written as Horn clauses.
• General form: A1 ∧ A2 ∧ ... ∧ An → H
• → Left side = Body (Antecedent)
• → Right side = Head (Consequent)
• Example: Student(x) ∧ happilyStudyingIn(x,y) → HappyStudent(x)
• If x is a student and studies happily, then x is a happy student.
• Constraints:
• Variables allowed (with x,y)
• Constants allowed (with ∧)
• Function symbols allowed 
• No classical negation
10

===== PAGE 10 =====
Where are the quantifiers? (universal? 
Existential?)
• Quantifiers are implicit. Variables in body implies universally 
quantified, while variables only in head implies existentially 
quantified
• isParent(X) → hasChild(X,Y) is equivalent to: 
• ∀X (isParent(X) → ∃Y hasChild(X,Y)). Meaning “Every parent has at 
least one child.”

===== PAGE 11 =====
Semantic Web Rule Language (SWRL): Rules + 
Ongologies integration
• SWRL is a rule language designed for OWL-DL (Description 
Logic). It allows Horn-like rules combined with OWL
• Example: 
• Person(?p) ∧ hasSibling(?p,?s) ∧ Man(?s) → hasBrother(?p,?s)
• If a person has a sibling who is a man, then that sibling is his brother.
• ?p, ?s are variables
• ∧ = logical AND
• → = implication

===== PAGE 28 =====
Word2Vec: Main idea
• Use a classifier to predict which words appear in the context of 
a target word (or vice versa)
• This classifier induces a dense vector representation of words 
(embedding)
• Words that appear in similar contexts (that have high distributional 
similarity) will have very similar vector representations.
• These models can be trained on large amounts of raw text

imply fact is false. We simply do not know.

Rules + Learning: hybrid approaches
• Hybrid reasoning integrates symbolic (logic-based) inference with 
statistical (embedding/neural) inference to create systems that are:
• explainable (symbolic rules, constraints)
• robust to noise (neural/embedding learning)
• generalizing (statistical models)

===== PAGE 79 =====
Symbolic → Guides → Embeddings
• Symbolic rules constrain or shape embedding training to ensure learned 
embeddings respect logical knowledge. 
•  Use rule regularization by adding rule-based constraints to the loss. L_total = 
L_embedding + λ * L_rule
• E.g. a horn rule: r1(x,y) ∧r2(y,z) → r3(x,z).
• Its embedding-based translation is: score_r1(x,y) * score_r2(y,z) ≤ score_r3(x,z)
• The penalty term is added: L_rule = max(0, score_r1(x,y)*score_r2(y,z) - score_r3(x,z))
• Use semantic regularizers to impose properties/characteristics
• E.g. for relations with symmetric property (such as marriedTo) you can define 
score(h, r, t) ≈ score(t, r, h)
• Some representative methods: 
• ComplEx-NNE+AER (embeddings + axiomatic constraints) 
• RUGE (iterative rule-guided embedding learning) 
• IterE (learning rules 
  embeddings iteratively)

Embeddings → Discover → Symbolic Rules
• Embeddings help discover new rules, such as property chains, type 
constraints, Horn rules, inverse relations, symmetric/asymmetric relations
•  if for many triples: the relation h + r1 + r2 ≈ t applies, then suggest r1(x,y) ^ 
r2(y,z) → r3(x,z)
• Benefits:
• Automates rule discovery (reduces expert workload) 
• Extracts emergent patterns from data
• Representative methods: 
• AMIE+ (rule mining; not neural but widely used) 
• Neural LP (learn logical rules with differentiable operators) 
• DRUM (differentiable rule mining through RNNs)
